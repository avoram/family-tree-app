
# Design System Rules

## Purpose

Define UI and design standards used throughout the application.

All user interfaces should follow these standards.

---

## Design System

- Use Angular Material components whenever possible.
- Prefer standard Material components over custom implementations.
- Follow Material Design guidelines.
- Maintain a consistent user experience across all screens.

---

## Layout Standards

- Design for desktop and mobile devices.
- Use responsive layouts.
- Avoid fixed-width layouts where possible.
- Maintain consistent spacing across pages.

---

## Form Standards

- Use Angular Material form controls.
- Provide clear labels for all fields.
- Display validation messages consistently.
- Mark required fields clearly.

---

## Navigation Standards

- Navigation should be simple and predictable.
- Similar actions should appear in consistent locations.
- Users should always know which family tree is currently selected.

---

## Tree Visualization Standards

- Family relationships should be easy to understand.
- Parent-child relationships should be visually clear.
- Spouse relationships should be visually distinct.
- Expand and collapse actions should be intuitive.
- Selected members should be clearly highlighted.

---

## Simplicity Principle

- Prefer clarity over visual complexity.
- Prefer existing Material components before creating custom components.
- Introduce custom UI components only when a business requirement cannot be satisfied using Angular Material.
